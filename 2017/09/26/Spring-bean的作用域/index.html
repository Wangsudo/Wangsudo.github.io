Spring-bean的作用域与自动装配

作者:Walker    发布于: 2017-09-26

bean的作用域

由配置文件中标签中scop属性确定的，若不设定，默认为单例

singleton单例

整个容器只有一个对象实例

prototype原型

每次获取bean都产生一个新的对象

request

每次请求时创建一个新的对象

session

在会话的范围内时的一个对象，这里指的是http中的session

global session

只在portlet下有用，表示是application

application

在应用范围中一个对象

注意

在整合struts2和spring的时候，要将action设为scope=“prototype”

自动装配-简化spring配置文件

在配置bean时，可以配置bean的autowire属性，用于指定装配类型。

no

不使用自动装配

byName

根据名称（set方法名来的）去查找相应的bean，如果有则装配上。

byType

根据类型进行自动装配，不用管bean的id，但是同一种类型的bean只能有一个，建议慎用。

constructor

当通过构造器注入 实例化bean时 适用byType的方式 装配构造方法

配置全局的自动装配类型

在头部配置default-autowire

建议

不使用自动分配而使用annotation

OVER

出门在外,在哪都是锻炼
